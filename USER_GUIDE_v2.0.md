# 거래명세서 입력 시스템 v2.0 사용 가이드

## 개요

태블릿에 최적화된 거래명세서 입력 앱입니다. 초성 검색, 내장 데이터베이스, 엑셀 내보내기 기능을 제공하여 Access 프로그램과 유사한 간소화된 회계 시스템을 구현했습니다.

---

## 주요 기능

### 1. 거래 입력 (입력 탭)

**초성 검색 기능:**
- 거래처명 입력 시 초성으로 검색 가능
  - 예: "ㅎㅂ" 입력 → "해비치" 검색
  - 예: "켈리" 또는 "7조" 입력 → "켈리케이(Kelly.K) // 7조" 검색
- 제품명 입력 시에도 초성 검색 지원
  - 예: "ㅅㅅㅅ" 입력 → "18색사" 검색

**다중 품목 입력:**
- 한 거래처에 여러 품목 동시 입력 가능
- "+ 품목 추가" 버튼으로 품목 추가
- 각 품목별로 독립적인 자동완성 제공

**취소/반품 처리:**
- 체크박스로 취소 또는 반품 표시 가능

**입력 순서:**
1. 거래처명 입력 (초성 검색 또는 직접 입력)
2. 품목명 입력 (초성 검색 또는 직접 입력)
3. 수량 입력
4. 필요 시 품목 추가
5. 저장하기 버튼 클릭

---

### 2. 거래 내역 조회 (내역 탭)

**날짜별 조회:**
- 날짜 선택으로 해당 일자의 거래 내역 조회
- "◀ 이전" / "다음 ▶" 버튼으로 날짜 이동
- "오늘로 이동" 버튼으로 빠른 이동

**통계 정보:**
- 총 거래 건수
- 총 수량 (취소/반품 제외)

**거래 내역 수정/삭제:**
- 각 거래 항목의 "수정" 버튼 클릭
- 거래처, 품목, 수량 수정 가능
- "삭제" 버튼으로 거래 내역 삭제

**엑셀 내보내기:**
- "엑셀 내보내기" 버튼 클릭
- 선택한 날짜의 거래 내역을 엑셀 파일로 저장
- 모바일: 공유 메뉴를 통해 이메일, 카카오톡, 구글 드라이브 등으로 전송 가능
- 웹: 자동으로 다운로드

---

### 3. 데이터 관리 (관리 탭)

**거래처 관리:**
- 새 거래처 추가
- 기존 거래처 삭제
- 별칭 지원 (예: "켈리케이" 와 "7조" 모두 검색 가능)

**제품 관리:**
- 새 제품 추가 (카테고리 선택: 여름실/겨울실/부자재)
- 기존 제품 삭제
- 카테고리별 분류

**구글 시트 설정 (선택 사항):**
- Google Apps Script Web App URL 입력
- 입력 시 거래 내역이 자동으로 구글 시트에 백업
- 설정하지 않아도 앱은 정상 작동 (내장 DB에만 저장)

---

## 데이터 저장 방식

### 주 저장소: 내장 데이터베이스
- 모든 거래 내역은 태블릿 내장 데이터베이스(SQLite)에 저장
- 인터넷 연결 불필요
- 빠른 속도
- 앱 내에서 조회/수정/삭제 가능

### 선택적 백업: 구글 시트
- 관리 탭에서 구글 시트 URL 설정 시 자동 백업
- 백업 실패해도 내장 DB 저장은 성공
- 기존 워크플로우와 호환

### 엑셀 내보내기
- 내역 탭에서 언제든지 엑셀 파일로 내보내기 가능
- 날짜별로 파일명 자동 생성 (예: `거래명세서_2026-02-06.xlsx`)

## 구글 시트 연동 설정 (선택 사항)

### 1. Google Apps Script 설정

1. [Google Sheets](https://sheets.google.com)에서 새 스프레드시트 생성
2. 상단 메뉴: **확장 프로그램 > Apps Script**
3. 다음 코드 복사 & 붙여넣기:

```javascript
function doPost(e) {
  try {
    const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    const data = JSON.parse(e.postData.contents);
    
    // 헤더가 없으면 추가
    if (sheet.getLastRow() === 0) {
      sheet.appendRow(['타임스탬프', '고객사 이름', '품목명', '수량', '취소, 반 반품']);
    }
    
    // 데이터 추가
    data.forEach(record => {
      sheet.appendRow([
        record.timestamp,
        record.customerName,
        record.productName,
        record.quantity,
        record.cancellation
      ]);
    });
    
    return ContentService.createTextOutput(JSON.stringify({
      success: true,
      message: 'Data saved successfully'
    })).setMimeType(ContentService.MimeType.JSON);
    
  } catch (error) {
    return ContentService.createTextOutput(JSON.stringify({
      success: false,
      error: error.toString()
    })).setMimeType(ContentService.MimeType.JSON);
  }
}
```

4. **배포 > 새 배포**
5. 유형: **웹 앱**
6. 액세스 권한: **모든 사용자**
7. **배포** 클릭
8. **웹 앱 URL** 복사

### 2. 앱에서 설정

1. 앱의 **관리** 탭 열기
2. **설정** 섹션으로 스크롤
3. "구글 시트 Web App URL" 입력란에 복사한 URL 붙여넣기
4. **저장** 버튼 클릭

이제 거래 내역 저장 시 자동으로 구글 시트에 백업됩니다.

## 배포 방법

### 웹 앱으로 배포 (추천)

1. 우측 상단 **"Publish"** 버튼 클릭
2. 생성된 URL을 태블릿 브라우저에서 접속
3. 브라우저 메뉴에서 **"홈 화면에 추가"** 선택
4. 이제 앱처럼 사용 가능

**장점:**
- 별도 설치 불필요
- iOS, 안드로이드, PC 모두 지원
- 배포 즉시 사용 가능
- 오프라인에서도 작동 (데이터는 기기에 저장)

- Apple Developer 계정 필요 ($99/년)
- 심사 기간: 1-3일

**Android (Google Play Store):**

## 자주 묻는 질문 (FAQ)

**Q: 인터넷 연결이 없어도 사용할 수 있나요?**
A: 네, 모든 데이터는 태블릿에 저장되므로 오프라인에서도 완벽하게 작동합니다. 구글 시트 백업만 인터넷이 필요합니다.

**Q: 데이터가 사라질 위험은 없나요?**
A: 앱을 삭제하지 않는 한 데이터는 영구 보존됩니다. 추가로 엑셀 내보내기나 구글 시트 백업을 사용하면 더 안전합니다.

**Q: 여러 태블릿에서 동시에 사용할 수 있나요?**
A: 각 태블릿은 독립적으로 작동합니다. 여러 기기 간 동기화가 필요하면 별도 문의 바랍니다.

**Q: 엑셀 파일은 어디에 저장되나요?**
A: 모바일에서는 공유 메뉴를 통해 원하는 곳(이메일, 드라이브 등)으로 전송할 수 있습니다. 웹에서는 다운로드 폴더에 저장됩니다.

**Q: 구글 시트 연동이 필수인가요?**
A: 아니요, 선택 사항입니다. 설정하지 않아도 앱은 정상 작동하며 내장 DB에 모든 데이터가 저장됩니다.
