# 구글 OAuth 설정 가이드

이 가이드는 거래명세서 앱에서 구글 시트 API를 사용하기 위한 OAuth 클라이언트 ID를 발급받는 방법을 안내합니다.

## 1단계: 구글 클라우드 콘솔 접속

1. 브라우저에서 [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 구글 계정으로 로그인

## 2단계: 새 프로젝트 생성

1. 상단의 **프로젝트 선택** 드롭다운 클릭
2. **새 프로젝트** 클릭
3. 프로젝트 이름 입력: `거래명세서 앱` (또는 원하는 이름)
4. **만들기** 클릭
5. 생성된 프로젝트 선택

## 3단계: Google Sheets API 활성화

1. 좌측 메뉴에서 **API 및 서비스** > **라이브러리** 클릭
2. 검색창에 `Google Sheets API` 입력
3. **Google Sheets API** 클릭
4. **사용 설정** 클릭

## 4단계: OAuth 동의 화면 구성

1. 좌측 메뉴에서 **API 및 서비스** > **OAuth 동의 화면** 클릭
2. **외부** 선택 후 **만들기** 클릭
3. 앱 정보 입력:
   - **앱 이름**: `거래명세서 앱`
   - **사용자 지원 이메일**: 본인 이메일
   - **개발자 연락처 정보**: 본인 이메일
4. **저장 후 계속** 클릭
5. **범위** 페이지에서 **범위 추가 또는 삭제** 클릭
6. 다음 범위 선택:
   - `https://www.googleapis.com/auth/spreadsheets`
   - `https://www.googleapis.com/auth/drive.file`
7. **업데이트** 클릭
8. **저장 후 계속** 클릭
9. **테스트 사용자** 페이지에서 **+ ADD USERS** 클릭
10. 본인 이메일 추가 후 **저장 후 계속** 클릭

## 5단계: OAuth 클라이언트 ID 생성

1. 좌측 메뉴에서 **API 및 서비스** > **사용자 인증 정보** 클릭
2. 상단의 **+ 사용자 인증 정보 만들기** 클릭
3. **OAuth 클라이언트 ID** 선택
4. 애플리케이션 유형: **웹 애플리케이션** 선택
5. 이름 입력: `거래명세서 웹 클라이언트`
6. **승인된 리디렉션 URI** 섹션에서 **+ URI 추가** 클릭
7. 다음 URI 추가:
   ```
   https://auth.expo.io/@your-username/transaction-record-app
   ```
   > **중요:** `your-username`을 본인의 Expo 사용자명으로 변경하세요.
   > Expo 사용자명을 모르는 경우, 터미널에서 `npx expo whoami` 실행

8. **만들기** 클릭
9. 팝업에서 **클라이언트 ID**를 복사하여 안전한 곳에 저장

## 6단계: 앱에 클라이언트 ID 설정

1. 프로젝트 루트에 `.env` 파일 생성 (이미 있다면 열기)
2. 다음 내용 추가:
   ```
   EXPO_PUBLIC_GOOGLE_CLIENT_ID=복사한_클라이언트_ID
   ```
3. 파일 저장

## 7단계: 앱 재시작

터미널에서 다음 명령어 실행:
```bash
pnpm dev
```

## 문제 해결

### "redirect_uri_mismatch" 오류
- 구글 클라우드 콘솔의 리디렉션 URI와 앱의 URI가 일치하는지 확인
- Expo 사용자명이 정확한지 확인

### "access_denied" 오류
- OAuth 동의 화면의 테스트 사용자에 본인 이메일이 추가되었는지 확인

### API 할당량 초과
- 무료 할당량: 일일 500회 읽기/쓰기
- 개인 사용에는 충분하지만, 초과 시 다음 날까지 대기 필요

## 추가 정보

- 구글 시트 API 문서: https://developers.google.com/sheets/api
- Expo AuthSession 문서: https://docs.expo.dev/versions/latest/sdk/auth-session/

## 보안 주의사항

- **클라이언트 ID는 공개되어도 안전**합니다 (프론트엔드에서 사용)
- **클라이언트 시크릿은 절대 공개하지 마세요** (서버에서만 사용)
- 이 앱은 클라이언트 ID만 사용하므로 시크릿은 필요 없습니다
