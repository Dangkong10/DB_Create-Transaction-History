# 거래명세서 입력 시스템 - 디자인 문서

## 프로젝트 개요

창고형 매장에서 거래처별 출고 내역을 실시간으로 입력하고 구글 시트에 자동 저장하는 태블릿 최적화 모바일 앱입니다. 고령 직원의 사용성을 최우선으로 고려하여 큰 글씨, 초성 검색, 빠른 입력 흐름을 제공합니다.

## 사용자 요구사항 분석

### 기존 문제점
1. **구글 폼의 스크롤 문제**: 거래처 목록이 길어 스크롤로 찾기 어렵고, 글씨가 작아 고령 직원이 사용하기 불편
2. **다중 품목 입력 불가**: 한 거래처가 3~6가지 물건을 가져갈 때 폼을 여러 번 제출해야 함
3. **바쁜 시간대 대응**: 손님이 몰릴 때 빠르게 입력하지 못해 대기 시간 발생
4. **OCR 오류**: 수기 작성 후 OCR 처리 시 오타로 인한 오류 발생

### 핵심 해결 방안
1. **초성 검색**: 'ㅎㅂ' 입력 시 '해비치', '한올' 등 자동 필터링
2. **다중 품목 입력**: 한 화면에서 여러 품목을 추가하고 한 번에 저장
3. **큰 UI 요소**: 고령 직원을 위한 큰 글씨(20px 이상), 큰 버튼, 넓은 터치 영역
4. **구글 시트 자동 저장**: 입력 즉시 구글 시트에 타임스탬프와 함께 저장

## 화면 구조

### 1. 홈 화면 (메인 입력 화면)
**주요 콘텐츠:**
- 거래처 검색 입력창 (초성 검색 지원)
- 품목 입력 폼 (동적 추가/삭제)
- 저장 버튼

**기능:**
- 거래처 이름 입력 시 초성 검색으로 자동완성 목록 표시
- 품목명/수량 입력 칸 (기본 1개, '+' 버튼으로 추가)
- 취소/반품 체크박스
- 저장 버튼 클릭 시 구글 시트에 전송

### 2. 설정 화면
**주요 콘텐츠:**
- 구글 시트 연동 설정
- 거래처 목록 관리
- 품목 목록 관리

**기능:**
- 구글 시트 API 키 입력
- 스프레드시트 ID 설정
- 거래처/품목 목록 동기화

## 주요 사용자 플로우

### 플로우 1: 단일 품목 입력
1. 사용자가 홈 화면 진입
2. 거래처 검색창에 초성 입력 (예: 'ㅎㅂ')
3. 자동완성 목록에서 '해비치' 선택
4. 품목명 입력 (예: '18색사')
5. 수량 입력 (예: '1')
6. '저장하기' 버튼 클릭
7. 구글 시트에 데이터 저장 완료 알림

### 플로우 2: 다중 품목 입력
1. 사용자가 홈 화면 진입
2. 거래처 검색창에 초성 입력 (예: 'ㅎㅂ')
3. 자동완성 목록에서 '해비치' 선택
4. 첫 번째 품목명/수량 입력 (예: '18색사', '1')
5. '+ 품목 추가' 버튼 클릭
6. 두 번째 품목명/수량 입력 (예: '24색사', '2')
7. '+ 품목 추가' 버튼 클릭
8. 세 번째 품목명/수량 입력 (예: '바늘', '1')
9. '저장하기' 버튼 클릭
10. 구글 시트에 3개 행으로 데이터 저장 완료 알림

### 플로우 3: 취소/반품 입력
1. 사용자가 홈 화면 진입
2. 거래처 검색창에 초성 입력
3. 자동완성 목록에서 거래처 선택
4. 품목명/수량 입력
5. '취소 및 반품' 체크박스 선택
6. '저장하기' 버튼 클릭
7. 구글 시트에 취소/반품 표시와 함께 데이터 저장

## 색상 선택

### 브랜드 컬러
- **Primary (강조색)**: `#2563EB` (파란색) - 버튼, 선택된 항목
- **Background (배경)**: `#FFFFFF` (밝은 모드), `#151718` (어두운 모드)
- **Surface (카드/입력창)**: `#F5F5F5` (밝은 모드), `#1E2022` (어두운 모드)
- **Foreground (주요 텍스트)**: `#11181C` (밝은 모드), `#ECEDEE` (어두운 모드)
- **Success (성공)**: `#22C55E` - 저장 완료 알림
- **Error (오류)**: `#EF4444` - 오류 메시지

### 색상 사용 가이드
- 저장 버튼: Primary 색상, 큰 크기로 화면 하단에 고정
- 입력창: Surface 색상, 테두리는 Border 색상
- 자동완성 목록: Surface 배경, 선택 시 Primary 색상 하이라이트
- 품목 추가 버튼: 연한 Primary 색상 배경

## 디자인 원칙

### 1. 접근성 우선
- **최소 글자 크기**: 20px (고령 직원을 위한 큰 글씨)
- **최소 터치 영역**: 48x48dp (애플 HIG 권장 사항)
- **명확한 대비**: WCAG AA 기준 이상의 색상 대비

### 2. 한 손 사용 최적화
- 주요 버튼은 화면 하단 1/3 영역에 배치
- 자주 사용하는 기능은 엄지손가락이 닿는 범위 내 배치
- 스크롤 최소화 (한 화면에 모든 입력 요소 표시)

### 3. 빠른 입력 흐름
- 최소한의 터치로 입력 완료
- 자동 포커스 이동 (거래처 선택 → 품목명 입력)
- 키보드 'Done' 버튼으로 입력 완료

### 4. 명확한 피드백
- 버튼 클릭 시 햅틱 피드백
- 저장 완료 시 성공 메시지 표시
- 오류 발생 시 명확한 오류 메시지

## 기술 스택

- **프레임워크**: React Native + Expo SDK 54
- **스타일링**: NativeWind (Tailwind CSS)
- **초성 검색**: 한글 자모 분리 라이브러리 (hangul-js 또는 자체 구현)
- **상태 관리**: React Context + useState
- **데이터 저장**: AsyncStorage (로컬), Google Sheets API (클라우드)
- **HTTP 클라이언트**: axios

## 구현 우선순위

1. **Phase 1**: 기본 UI 구조 및 레이아웃
2. **Phase 2**: 초성 검색 라이브러리 통합
3. **Phase 3**: 다중 품목 입력 기능
4. **Phase 4**: 구글 시트 API 연동
5. **Phase 5**: 태블릿 최적화 및 테스트
