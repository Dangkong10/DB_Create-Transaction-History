# 거래명세서 입력 시스템 — 사용 가이드

창고형 매장에서 거래처별 출고 내역을 빠르고 정확하게 입력하기 위한 태블릿 최적화 PWA입니다. Supabase 클라우드 DB와 연동되어 여러 기기에서 실시간 동기화됩니다.

---

## 주요 기능

- **초성 검색**: `ㅎㅂ` 입력만으로 `해비치` 검색
- **다중 품목 입력**: 한 거래처의 여러 품목을 한 번에 입력
- **취소/반품 처리**: 체크박스 하나로 반품 건 표시
- **내역 조회**: 날짜별 거래 내역 조회·수정·삭제
- **엑셀 내보내기**: 선택한 날짜의 거래 내역을 `.xlsx` 파일로 저장
- **오프라인 동작**: 네트워크가 끊겨도 로컬(IndexedDB)에 저장, 온라인 복귀 시 자동 동기화
- **클라우드 백업**: 저장된 거래 내역은 Supabase에 자동 업로드, 여러 기기에서 공유

---

## 설치 (PWA)

별도 앱 설치 파일 없이 브라우저에서 "홈 화면에 추가"로 앱처럼 사용합니다.

### iOS (iPad/iPhone, Safari)
1. Safari에서 관리자가 안내한 앱 주소에 접속
2. 하단 **공유 버튼(네모+화살표)** 탭
3. **"홈 화면에 추가"** 선택
4. 홈 화면의 아이콘으로 실행

### Android (Chrome)
1. Chrome에서 앱 주소에 접속
2. 우측 상단 **⋮ 메뉴** → **"홈 화면에 추가"** 또는 **"앱 설치"**
3. 홈 화면 아이콘으로 실행

### PC (Chrome/Edge)
1. 브라우저 주소창 우측의 **설치 아이콘** 클릭
2. **"설치"** 선택

---

## 로그인

- 관리자가 발급한 **이메일과 비밀번호**로 로그인합니다.
- 로그인 후에는 브라우저를 닫아도 세션이 유지됩니다.
- 세션이 만료되거나 로그아웃되면 자동으로 로그인 화면으로 이동합니다.

---

## 사용 방법

### 1) 거래 입력 (입력 탭)

1. **거래처 이름** 입력창에 이름 또는 초성 입력 (예: `ㅎㅂ` → 해비치)
2. 자동완성 목록에서 선택
3. **품목명** 입력 (초성 검색 지원)
4. **수량** 입력
5. 품목이 여러 개면 **"+ 품목 추가"** 로 행 추가
6. 취소·반품 건이면 체크박스 선택
7. **저장하기** 버튼 클릭

> 💡 네트워크 상태와 무관하게 저장 버튼은 즉시 성공합니다. 오프라인 상태일 때는 로컬에 쌓였다가 온라인 복귀 시 자동으로 Supabase에 동기화됩니다.

### 2) 내역 조회 (내역 탭)

- **날짜 선택 / ◀ 이전 / 다음 ▶ / 오늘로 이동** 으로 일자 전환
- 총 거래 건수·총 수량 요약 표시 (취소·반품 제외)
- 항목의 **수정** / **삭제** 버튼으로 편집
- **엑셀 내보내기** 버튼: 해당 날짜의 거래 내역을 `거래명세서_YYYY-MM-DD.xlsx` 로 다운로드 (모바일은 공유 메뉴로 전송)

### 3) 관리 (관리 탭)

- **거래처 관리**: 거래처 추가·삭제, 별칭 설정 (예: `켈리케이`와 `7조` 모두 검색되도록)
- **품목 관리**: 여름실 / 겨울실 / 부자재 카테고리별 추가·삭제
- **특가 관리**: 거래처별 품목 특별 가격 설정

## 데이터 저장 방식

| 계층 | 저장소 | 역할 |
|---|---|---|
| 로컬 | IndexedDB (브라우저 내장) | 오프라인 저장, 동기화 대기열(syncQueue) |
| 클라우드 | Supabase (PostgreSQL + RLS) | 여러 기기 간 동기화, 영구 보관 |

- **오프라인 저장**: 네트워크가 없으면 IndexedDB에 저장되고 `syncStatus: pending` 으로 표시됩니다.
- **자동 동기화**: 온라인 복귀 시 30초 간격으로 Supabase에 자동 업로드합니다.
- **사용자 격리**: 모든 데이터는 로그인한 사용자 본인만 조회·수정할 수 있습니다 (RLS).

## FAQ

**Q. 인터넷이 끊겨도 쓸 수 있나요?**
네. 저장은 로컬 IndexedDB에 먼저 기록되고 네트워크 복귀 시 자동으로 Supabase에 올라갑니다.

**Q. 여러 태블릿에서 같은 계정으로 쓰면 데이터가 공유되나요?**
네. Supabase가 중앙 저장소이므로 같은 이메일로 로그인하면 모든 기기에서 같은 내역이 보입니다.

**Q. 거래처가 검색되지 않아요.**
초성이 정확한지 확인하세요. 거래처 별칭을 추가하면 여러 이름으로 검색됩니다 (관리 탭에서 설정).

**Q. 저장한 내역을 수정할 수 있나요?**
내역 탭에서 항목의 **수정** 버튼으로 가능합니다. 취소·반품 건은 별도 체크박스 사용을 권장합니다.

**Q. 엑셀 파일은 어디로 저장되나요?**
PC: 브라우저 다운로드 폴더. 모바일: 공유 메뉴를 통해 드라이브·이메일·메신저 등으로 전송 가능.

**Q. 로그인이 풀리면 저장 중이던 내용은 어떻게 되나요?**
세션 만료가 감지되면 로그인 화면으로 이동하며, 아직 업로드되지 않은 대기 내역은 확인 후 정리됩니다.

## 문제 해결

- **앱이 실행되지 않음**: 브라우저 캐시 삭제 후 재실행, 또는 재설치
- **입력이 느림**: 태블릿 재시작, 백그라운드 앱 정리
- **키보드가 화면을 가림**: 입력창 위로 스크롤하거나 키보드 닫기(완료)
- **동기화가 안 됨**: 인터넷 연결 확인 → 네트워크 복귀 후 30초 내 자동 업로드. 계속 실패 시 로그아웃 후 재로그인

## 기술 스택

- **프레임워크**: Expo (React Native for Web) + Expo Router
- **언어**: TypeScript
- **스타일**: NativeWind (Tailwind CSS)
- **로컬 저장**: IndexedDB (오프라인 큐 포함)
- **클라우드**: Supabase (Auth + PostgreSQL + RLS)
- **엑셀**: `xlsx`, `exceljs`
- **배포 형태**: PWA (홈 화면에 추가하여 독립 앱처럼 사용)
